# Bitácora del Ingeniero

Hace muchos años aprendí a programar utilizando BASIC.

Como muchos ingenieros de mi generación, escribía programas pensando principalmente en resolver un problema específico: realizar cálculos, automatizar tareas o procesar información.

Con el tiempo llegaron otros lenguajes, nuevas metodologías y proyectos mucho más grandes. Descubrí que aprender un lenguaje no consiste únicamente en memorizar instrucciones, sino en comprender cómo ese lenguaje ayuda a organizar ideas y construir soluciones mantenibles.

Cuando conocí Python entendí que era diferente.

No intentaba impresionar al programador con una sintaxis compleja; buscaba que el código fuera claro, legible y cercano al lenguaje humano.

Años después, trabajando con QGIS, PostGIS, GeoServer y aplicaciones para la gestión del agua, confirmé que esa filosofía era una de las razones por las que Python se convirtió en el lenguaje dominante dentro del mundo geoespacial.

Este libro nace precisamente con esa idea.

No aprenderemos únicamente un lenguaje de programación.

Aprenderemos una forma de pensar y de construir software profesional aplicado a los Sistemas de Información Geográfica.

---

# ¿Por qué este libro?

Existen cientos de libros sobre Python.

Entonces, ¿por qué escribir uno más?

Porque la mayoría están orientados a enseñar el lenguaje de forma general.

Los ejemplos suelen centrarse en:

- listas de frutas;
- nombres de personas;
- cuentas bancarias;
- estudiantes;
- juegos sencillos.

Todos son útiles para aprender la sintaxis, pero pocas veces reflejan los problemas que enfrenta un profesional de los Sistemas de Información Geográfica.

En este libro utilizaremos ejemplos relacionados con:

- ingeniería del agua;
- cartografía;
- sensores;
- drones;
- imágenes satelitales;
- bases de datos espaciales;
- servicios OGC;
- QGIS;
- PostGIS;
- GeoServer.

Nuestro objetivo no es aprender Python de forma aislada.

Nuestro objetivo es aprender Python para desarrollar soluciones geoespaciales reales.

---

# 🎯 ¿Por qué estoy aprendiendo esto?

Porque todas las bibliotecas que utilizaremos durante el libro fueron diseñadas siguiendo la filosofía de Python.

Comprender esa filosofía hará que aprender nuevas herramientas resulte mucho más sencillo.

# ¿Por qué Python?

Python fue creado con una idea muy clara.

> El código debe ser fácil de leer.

Esta filosofía parece sencilla, pero cambió la historia de la programación.

Actualmente Python impulsa proyectos en:

- Ciencia de Datos.
- Inteligencia Artificial.
- Robótica.
- Automatización.
- Computación en la nube.
- Desarrollo Web.
- Sistemas de Información Geográfica.

Su éxito no se debe únicamente a la cantidad de bibliotecas disponibles.

Se debe principalmente a que permite concentrarse en resolver problemas, no en luchar contra la sintaxis del lenguaje.

# Nuestro método de aprendizaje

Este libro seguirá una metodología diferente.

No memorizarás comandos.

Comprenderás cómo funciona Python.

Cada nuevo concepto responderá siempre cinco preguntas:

1. ¿Qué es?
2. ¿Por qué existe?
3. ¿Cómo funciona?
4. ¿Cuándo debe utilizarse?
5. ¿Cómo se aplica en QGIS?

Nuestro objetivo es formar desarrolladores capaces de aprender cualquier biblioteca Python de manera autónoma.

---

# Principios del libro

A lo largo de toda la obra seguiremos estas reglas:

- Comprender antes que memorizar.
- Escribir código limpio.
- Explicar el "por qué" además del "cómo".
- Relacionar cada concepto con un problema real.
- Construir conocimiento de manera progresiva.
